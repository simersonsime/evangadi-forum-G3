Backend Commands Reference - for Evangadi Forum project
1. Project Initialization
- npm init -y  - Initialize a new Node.js project with default settings.
- npm install  - Install all dependencies from package.json

2. Install Core Packages
- npm i express mysql2 dotenv
    - express → Web server & API routing
    - mysql2 → MySQL driver
    - dotenv → Environment variable management
- npm i jsonwebtoken - JWT for authentication
- npm install bcryptjs - Securely hash passwords
- npm install --save-dev nodemon - Auto-restart server during development

3. Common npm Commands
- npm install          # Install dependencies
- npm uninstall <pkg>  # Remove a package
- npm update           # Update all packages
- npm list --depth=0   # List installed packages
- npm audit            # Check vulnerabilities
- npm audit fix        # Auto-fix vulnerabilities

4. Running Server
- Standard Node.js: node server.js
- With nodemon: npx nodemon server.js (auto-reload during development):

5. Backend Folder Structure
backend/
├── config/        # db.js → MySQL connection
├── controllers/   # authController.js, questionController.js, answerController.js
├── middleware/    # authMiddleware.js → JWT verification
├── models/        # userModel.js, questionModel.js, answerModel.js, tables.sql
├── routes/        # authRoutes.js, questionRoutes.js, answerRoutes.js
├── utils/         # generateToken.js → JWT helper
├── app.js         # Express setup
├── server.js      # Start server
├── .env           # Environment variables
├── package.json
└── package-lock.json

6. Workflow Summary
  1. Initialize project → npm init -y
  2. Install dependencies → npm install express mysql2 dotenv jsonwebtoken bcryptjs
  3. Setup .env file
  4. Create MySQL database & tables
  5. Configure db.js
  6. Build controllers, routes, and JWT middleware
  7. Start server → node server.js or npx nodemon server.js
  8. Test APIs with Postman
